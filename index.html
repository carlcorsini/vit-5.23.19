Git Cheat Sheet

Goal: Create a project using the terminal and connect it with our github

Git Refresher

To begin create a new directory ($ mkdir day-3) using the terminal, change into that directory ($ cd day-3) and create an index.html file ($ touch index.html)

Use the $ code . command to open your project in VScode

In your index.html file use the ! shortcut to create html boiler plate and save

Now that you have created a new project we are ready to push to to the cloud!

These steps are broken down in more detail in the 5.22.19 Recap

Navigate back to the terminal and ensure you are in the new project directory ($ pwd) and run the command:

$ git status

Hopefully you'll receive an error:


If not don't fret! There are 2 other messages you may receive:

1. Command not found

2. Some sort of message about the status of your branch:


If you receive any of these messages, there is an easy fix, see me at some point during class and we'll get git fixed


If you received a fatal error from step 1 the next step is to initialize git with the command:

$ git init

We have created a repository on your local machine

use the command: $ git status to check on the status of your local repository and you should receive this message:

This message is showing us files that are not tracked yet by git

add the files to the staging area using the command:

$ git add index.html or $ git add . (essentially git add all)

then check your git status again:

$ git status

now your tracked files should be green, they are ready to be committed to the repository

Almost there! Now we will commit or take a snapshot of our project at this point with the command:

$ git commit -m "first commit"

We took at snapshot of the code, now we will link this local repository with one the cloud

Navigate to github.com and create a new repository

click the "New" button

On the next page DO NOT initialize the repository with a README, and name it "day-3" (or whatever you'd like)

click create repository

Next we will copy the "...or push an existing repository from the command line" commands one by one


FIRST: $ git remote add origin https://github.com/USERNAME/REPOSITORYNAME.git

WAIT now make a change in your index.html file, save and commit those changes, refer to steps 4 and 5

Proceed with the last command:

SECOND: $ git push -u origin master


You may be asked for your github credentials, type them in and you should see the above message. You may also receive an authentication error, that means you entered the incorrect credentials, see me for help on this step

Navigate back to github, refresh the page and you'll see your project now exists on github!

If you ran in to errors at any point in this process don't worry, if you were able to connect your project to github give yourself a round of applause!
